TypeScript helps us to write better code. It achieves this by forcing developers to comply with strict data types. In TypeScript, data types are essential for defining the kind of values that variables can hold, helping developers catch errors early in the coding process. TypeScript offers several built-in data types, including primitive types like number, string, boolean, null, and undefined. Additionally, it supports more complex types such as arrays, tuples, and enums. By specifying data types, developers can ensure that variables are used consistently, making the code easier to read and maintain.

Moreover, TypeScript allows for user-defined types through interfaces and type aliases. This flexibility enables developers to create custom data structures tailored to their application’s needs, promoting better organization and type safety. With TypeScript's strong data typing system, us programmers can take advantage of powerful features like autocompletion and type checking in our IDEs (such as NeoVim, or VSCode), which lead to fewer runtime errors and a more robust development experience. Overall, understanding data types is crucial for leveraging TypeScript’s capabilities effectively.

Not only does TypeScript help us write better code, but it also bridges the gap between JavaScript and other programming languages that support similar features. Over time, it seems that some JavaScript developers have become complacent, resulting in poorly written and sloppy code. Code that, if written in any other compiled language, would raise questions about a developer's competency.
